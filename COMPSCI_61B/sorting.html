1. Sorting

Sorting solves the problem of putting a set of elements in order. To put a set of elements in order, there must be some relation \(<\) that we can define for those elements that satisfy two conditions:

  1. Law of Trichotomy: Exactly one of \(a < b\), \(a = b\), or \(b < a\) is true.
  2. Law of Transitivity: If \(a < b\) and \(b < c\), then \(a < c\).

An inversion is a pair of elements that are out of order with respect to our relation \(<\).

1.1. Selection Sort

To selection sort \(N\) items, we:

  1. Find the smallest item in the unsorted portion of the array.
  2. Swap it to the end of the sorted portion of the array.
  3. Selection sort the remaining unsorted items.

The time complexity of selection sort is \(N + (N-1) + \cdots + 1 = \Theta(N^2)\), and the space complexity is \(\Theta(1)\) because we do not need extra memory.

1.2. Heap Sort

Selection sort is inefficient because we have to search the entire unsorted array for the smallest item every time. The fix is to first insert all the items into a max heap.

To heap sort \(N\) items, we can then:

  1. Delete largest item from the max heap.
  2. Put deleted item at the first open slot starting from the end of our output array.

The time complexity of heap sort is \(O(N \log N)\). The space complexity of heap sort is the memory we need to build the copy of our data, so it is \(\Theta (N)\).

1.2.1. In-Place Heap Sort

To increase memory efficiency, instead of using a separate output array, we can use the same input array and sort in-place. First, we can create a complete binary tree directly using the elements in our input array, and then “heapify” the result. To do this, we:

  1. Sink nodes in reverse level order.
  2. After sinking, we guarantee that the tree rooted at that node is a heap.

[interactive demo of heapification here]

Now that we have our heap, we can run heap sort in-place like so:

  1. Delete largest item from the max heap by swapping the root with the last item in the heap.
  2. Sink until the heap is valid again.
  3. Repeat for all \(N\) items.

Now, our space complexity is just \(\Theta(1)\) since we don’t need any extra memory.

1.3. Merge Sort

To merge sort \(N\) items, we:

  1. Split items into two roughly even pieces.
  2. Merge sort each half recursively.
  3. Merge the two sorted halves to form the final result
